Allocation of memory for local objects
    at compile time the exact ammount of memory is known for the local variables and is statically allocated
    code stored in program block
local vars, objects, and temporary objects are allocated memory on the call stack
    - fixed and does not change during runtime 
    - allocated on the stack when function is invoked and de-allocated when function ends

Dynamically allocated memory:
    allocated on the heap- 
        - managed by the operating system
    blocks of memory can be allocated and deallocated to a program
    'new' operator used to allocate memory to the heap
    'delete' or 'delete[]' operators used to deallocate memory on the heap 

    // new returns an adress of a memory location on the heap
int *ptr = new int;   //ptr points to an int
        ptr = 0;        // memory leak, can no longer acess previous dynamic int
*ptr = 10;            // the dynamic int val is now 10
delete ptr;           // ptr released
*ptr = 20;            // dangling pointer / error
    should set the ptr = 0 after deallocation to avoid errors

Allocating Arrays Dynamically:
    int n = 5;
    int *ptr = new int[n];
    *ptr = 10;                  // ptr[0] = 10
    *(ptr + 1) = 20;            // ptr[1] = 20
    *(ptr + 2) = 30;            // ptr[2] = 30
can also use bracket operator
    ptr[0] = 1;                 // ptr[2] = 1
    ptr[1] = 2;                 // ptr[2] = 2
    ptr[2] = 3;                 // ptr[2] = 3
